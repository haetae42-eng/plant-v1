# 정원탭 고정 스펙 문서 (v1.0)

## 1) 이 문서는 왜 필요한가
- 정원탭을 만들 때 기준이 계속 바뀌면, 같은 수정을 반복하게 됩니다.
- 이 문서는 `디자인 기준 + 동작 기준`을 고정해서, 이후 수정 요청을 빠르게 처리하기 위한 기준표입니다.
- 비유하면 인테리어 공사 전 도면입니다.

## 2) 적용 범위
- 탭: `정원`
- 기준 화면: 모바일 세로 화면 (`390 x 676`)
- 목적: 수집한 식물을 깔끔하게 전시하고, 페이지로 넘겨서 관리하는 기능

## 3) 레이아웃 고정값

### 3-1. 전시 슬롯 구조
- 배치 구조: `2-3-2`
- 열 이름: `A열 / B열 / C열`
- 슬롯 수: 페이지당 7칸
- 페이지 수: 3페이지

### 3-2. 슬롯 좌표 (현재 코드 기준)
| 열 | 슬롯 ID | X | Y |
|---|---|---:|---:|
| A열 | display-top-1 | 116 | 214 |
| A열 | display-top-2 | 274 | 214 |
| B열 | display-mid-1 | 76 | 386 |
| B열 | display-mid-2 | 195 | 386 |
| B열 | display-mid-3 | 314 | 386 |
| C열 | display-bottom-1 | 116 | 558 |
| C열 | display-bottom-2 | 274 | 558 |

### 3-3. 페이지 UI
- 하단 페이지 버튼: `이전 / 현재페이지 / 다음`
- 현재 페이지 표기: `1 / 3` 형식
- 첫 페이지에서 `이전` 비활성, 마지막 페이지에서 `다음` 비활성

## 4) 상태 규칙 (어떤 상태가 있는지)
- 빈 슬롯: `+` 아이콘 + `빈 슬롯` 표시
- 배치됨: 화분 + 식물 + 이름표 표시
- 샘플 ON: 테스트용 픽셀 표현 모드
- 샘플 OFF: 기본(일러스트) 표현 모드

## 5) 행동 규칙 (유저가 누르면 어떻게 되는지)

### 5-1. 자동 배치
- 식물을 새로 획득하면 정원에 자동 배치 시도
- 순서: 도감 획득 순서대로
- 배치 위치: 가장 먼저 비어 있는 슬롯

### 5-2. 중복 배치 금지
- `식물 1종류당 정원 전체(모든 페이지)에서 최대 1개만 배치`
- 이미 배치된 식물은 다른 슬롯에 중복 배치 불가

### 5-3. 재고 규칙
- 전시 재고 값은 `0 또는 1`만 사용
- 배치하면 재고 `1 -> 0`
- 회수하면 재고 `0 -> 1`

### 5-4. 수동 배치/회수
- 빈 슬롯 클릭: 선택한 식물 배치 시도
- 배치 슬롯 클릭: 회수 시도
- 실패 케이스:
  - 배치할 식물 미선택
  - 배치할 화분 미선택
  - 재고 없음
  - 이미 같은 식물이 다른 슬롯에 배치됨

## 6) 시각 규칙
- 이름표는 식물명(닉네임 우선, 없으면 기본 이름) 표시
- 식물과 화분은 항상 같은 슬롯 중심 정렬
- 픽셀 샘플 모드에서도 이름표는 동일 위치 규칙 유지
- 빈 슬롯 `+`는 가독성 우선(초록 계열)

## 7) 데이터 규칙
- 슬롯 저장 단위: `slotId + layer + page + potId + speciesId`
- 페이지별 슬롯 목록을 저장/복원
- 저장 로드 시 잘못된 값 보정:
  - 페이지 범위 벗어나면 제거
  - 중복 배치 식물은 1개만 유지
  - 소유하지 않은 화분 ID면 기본 화분으로 보정

## 8) 에셋 규칙 (파일 넣을 때 기준)

### 8-1. 일반 정원 식물
- 기본 소스: `app/public/assets/plants/...`

### 8-2. 권장 제작 기준
- 캔버스: `96x96` 또는 `128x128`
- 중요 요소(꽃 머리/잎/줄기)는 캔버스 중앙에 밀도 있게 배치
- 투명 여백이 너무 크면 게임에서 작아 보임

## 9) QA 체크리스트 (수정 후 꼭 확인)
- 정원 페이지가 1~3까지 정상 이동되는지
- 2-3-2 위치가 깨지지 않는지
- 빈 슬롯 `+`가 모든 빈 칸에 보이는지
- 같은 식물 중복 배치가 막히는지
- 식물 회수 시 재고가 1로 돌아오는지
- 새로고침 후 배치 상태가 유지되는지
- 샘플 ON/OFF에서 레이아웃이 무너지지 않는지
- 이름표가 잘리거나 겹치지 않는지

## 10) 변경 원칙
- 이 문서를 기준으로 개발합니다.
- `레이아웃/크기/상태/동작` 변경은 문서 먼저 업데이트 후 코드 반영합니다.

## 11) 픽셀 디자인 시스템 (이번에 적용한 기본값)

### 11-1. 식물 사이즈 3등급
- `small`: 작은 꽃/줄기형
- `medium`: 기본형
- `large`: 잎/꽃 볼륨이 큰 식물

기본 매핑(현재):
- `small`: 튤립
- `medium`: 금잔화, 커피나무, 스투키, 청옥
- `large`: 장미, 몬스테라

### 11-2. 이름표 디자인 2안
- `rect`: 직사각 이름표
- `round`: 둥근 라운드 이름표

현재 기본값:
- `round`

### 11-3. 빈 슬롯 `+` 아이콘 2안
- `line`: 글자형 `+`
- `block`: 도형형 `+` (굵고 가독성 높음)

현재 기본값:
- `block`

### 11-4. 화분 공통 팔레트
- 픽셀 샘플 모드에서 화분은 공통 4톤으로 통일:
  - 림
  - 몸통
  - 그림자
  - 흙

목적:
- 식물 색감이 더 돋보이게 하고, 화면 전체 톤을 정리하기 위함
