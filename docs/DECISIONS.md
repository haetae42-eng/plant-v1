# 결정 로그

최종 업데이트: 2026-03-13

## D-001 정원 전시 보상 위치
- 결정:
  - 정원 탭 직접 보상 UX는 축소
  - 핵심 수령 루프는 도감 `세트보상`으로 이동
- 이유:
  - 보상 채널 과다로 인한 경제 과보상 방지
  - 유저가 보상 위치를 한 곳에서 인지하도록 단순화
- 영향:
  - 정원은 “배치/편집/감상” 중심
  - 도감은 “완성/수령” 중심

## D-002 도감 카테고리 구조
- 결정: `꽃 / 식물 / 세트보상`
- 이유:
  - 기존 다육 분리보다 현재 에셋 상태와 UX 의도가 명확
- 영향:
  - 세트 카드에서 코인+씨앗 수령 가능
  - 하단 도감 탭에 수령 가능 빨간점 노출

## D-003 정원 시각 스타일
- 결정:
  - 정원은 카툰 모드 고정
  - 픽셀 토글은 정원에서 제거(렌더 분기 비활성)
- 이유:
  - 탭별 스타일 정체성 분리
  - 사용자 요청: 정원은 귀엽고 아기자기한 카툰 감성 우선
- 영향:
  - 정원 렌더 경로 단순화
  - 픽셀 샘플 자산 의존 감소

## D-004 정원 편집 UX
- 결정:
  - 슬롯 탭 즉시 편집 모달 오픈
  - 모달에서 식물/화분 변경 후 저장
  - 닫기 방식은 우상단 X 버튼 중심
- 이유:
  - 흐린 오버레이 채널보다 모달이 명확하고 모바일 친화적
- 영향:
  - `isOpeningDecorSlotEditModal` 가드 도입
  - 슬롯 클릭 플로우 단순화

## D-005 초반 7일 루프
- 결정:
  - 출석 보상과 별도로 `초반 7일 해금 보상`을 명시적으로 노출/지급
- 이유:
  - 초반 동기 강화 및 해금 체감 개선
- 영향:
  - `attendance.unlockClaimedDays` 상태 관리 추가
  - 출석 UI 안내 문구/요약 카드 강화

## D-006 상단/하단 탭 톤
- 결정:
  - 상단 리소스/하단 탭은 기존 톤으로 복구
  - 과한 탭형 리디자인은 채택하지 않음
- 이유:
  - 사용성/가독성 피드백 반영
- 영향:
  - 색상/정렬 중심 소폭 개선 유지

## D-007 로컬 개발 포트 운영
- 결정: 팀 작업 기준 포트 `5174` 사용
- 이유:
  - 모바일 실기기 확인 흐름과 기존 운영 스크립트 일치
- 영향:
  - `start_server.command` 기준으로 서버 관리

## D-008 정원 상단 우측 버튼 역할
- 결정:
  - `편집` 버튼을 `비우기` 토글로 전환
  - `비우기ON` 상태에서 슬롯 탭 시 즉시 회수
- 이유:
  - 슬롯이 꽉 찬 상황에서 빈 슬롯 UX를 바로 만들기 어렵다는 사용자 피드백 반영
  - 모달 진입 없이 빠른 회수 동선 제공
- 영향:
  - 정원 탭 상단 버튼 라벨/색상 상태값 변경
  - 비우기 모드에서 편집 모달 오픈 차단

## D-009 저장 토스트 정책
- 결정:
  - 저장 성공 문구는 `저장 완료`로 고정
  - 저장 실패 문구는 짧은 이유만 표시
- 이유:
  - 긴 성공 문구는 사용자 체감 효용이 낮고 가독성이 떨어짐
- 영향:
  - 편집 모달 저장 피드백 단순화
  - 실패 케이스 원인 파악은 유지

## D-010 플레이테스트용 초기화 정책
- 결정:
  - 테스트 강제 코인/해금 로직 제거
  - 테스트 시기 세이브와 실사용 테스트 세이브를 분리(`SAVE_STORAGE_KEY v2`)
- 이유:
  - 주변 사용자 실제 플레이 테스트 시 경제/해금 검증 왜곡 제거
- 영향:
  - 기존 테스트 세이브 자동 배제
  - 신규 세이브는 기본 상태(비강제 해금)로 시작

## D-011 세트카드 필요 종 표기 규칙
- 결정:
  - 세트카드 `필요 :` 문구는 전체 필요 종 이름을 직접 노출
  - 1줄 3종 + 2줄 나머지 종으로 고정 표시
- 이유:
  - `외 N종` 표기만으로는 사용자에게 필요한 종이 명확하지 않음
- 영향:
  - 세트 진행 목표 가독성 향상
